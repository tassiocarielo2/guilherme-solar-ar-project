1. Introdução

Este projeto utiliza MindAR.js para criar uma experiência de Realidade Aumentada com planetas do sistema solar. Se você está enfrentando problemas com a detecção de marcadores, este guia fornecerá soluções e dicas para melhorar a experiência.

Dica: Para que a AR funcione corretamente, você precisa de:
  • Acesso via HTTPS (necessário para acesso à câmera)
  • Iluminação adequada para visualizar os marcadores
  • Marcadores com boa qualidade e contraste
  • Um dispositivo com câmera e suporte a WebGL

2. Solucionando Problemas de Detecção

Problemas Comuns e Soluções

Problema Possível Causa Solução
Câmera ativada mas não detecta marcadores Arquivo .mind vazio ou inválido Regenerar o arquivo .mind usando o gerador fornecido
Erro ao iniciar o sistema AR Acesso à câmera negado ou HTTPS não disponível Garantir permissões de câmera e acessar via HTTPS
Detecção intermitente Iluminação inadequada ou marcador de baixa qualidade Melhorar iluminação e usar marcadores com mais contraste
Modelo 3D não aparece Problema ao carregar o modelo GLB Use o botão "Alternar Modelo 3D" para usar o modelo de fallback

Verificações Básicas

  1. Certifique-se de que o arquivo jupiter.mind existe em /targets e não está vazio
  2. Verifique se há mensagens de erro no console do navegador (F12 ou Inspecionar)
  3. Teste o botão "Testar Detecção" para verificar se o modelo e as informações aparecem corretamente
  4. Teste a página test-marker.html para verificar se a detecção funciona com marcadores conhecidos

3. Criando Bons Marcadores

O reconhecimento AR depende muito da qualidade dos marcadores. Ao fotografar os planetas de isopor, siga estas diretrizes:

Características de Bons Marcadores

  • Alto contraste - Diferença clara entre áreas claras e escuras
  • Rico em detalhes - Muitos padrões, texturas e características identificáveis
  • Assimetria - Padrões que não se repetem exatamente
  • Bordas nítidas - Contornos bem definidos
  • Sem reflexos - Evite superfícies brilhantes que causem reflexos

Marcador ruim: pouco contraste, poucos detalhes

Marcador bom: alto contraste, muitos detalhes

Dicas para Fotografar Planetas de Isopor

  1. Use boa iluminação (luz natural ou difusa)
  2. Evite reflexos diretos na superfície
  3. Pinte detalhes com alto contraste (ex: manchas, anéis, etc.)
  4. Fotografe contra um fundo de cor neutra
  5. Capture de múltiplos ângulos para escolher a melhor imagem
  6. Use uma resolução média (600-1200px) - nem muito grande nem muito pequena
  7. Garanta que a imagem esteja nítida e focada
Atenção: Superfícies muito uniformes ou com padrões repetitivos não funcionam bem como marcadores. Assegure-se de pintar características distintas em cada planeta.

4. Arquivos .mind

Os arquivos .mind contêm os dados processados das imagens para reconhecimento. Existem algumas formas de criá-los:

Opção 1: Gerador Online

Use a página generate-mind-file.html incluída neste projeto para criar seu arquivo .mind diretamente no navegador.

Opção 2: Compilador Oficial MindAR

  1. Acesse https://hiukim.github.io/mind-ar-js-doc/tools/compile/
  2. Faça upload das imagens dos planetas
  3. Clique em "Compile" e aguarde o processamento
  4. Baixe o arquivo .mind gerado
  5. Coloque o arquivo na pasta targets/jupiter.mind

Opção 3: Usar Marcadores de Exemplo Temporariamente

Se você estiver com dificuldades, pode temporariamente usar marcadores conhecidos para testar:

// No index.html, mude a linha:
mindar-image="imageTargetSrc: targets/jupiter.mind; ..."

// Para:
mindar-image="imageTargetSrc: https://cdn.jsdelivr.net/gh/hiukim/mind-ar-js@1.2.1/examples/image-tracking/assets/card-example/card.mind; ..."

5. Testando o Sistema

Para uma experiência completa de testes, siga esta sequência:

  1. Primeiro, teste a página test-marker.html com os marcadores de exemplo para confirmar que a detecção básica funciona no seu dispositivo
  2. Se os marcadores de exemplo funcionarem, gere seu próprio arquivo .mind usando o generate-mind-file.html
  3. Coloque o arquivo gerado em targets/jupiter.mind
  4. Teste a aplicação principal (index.html) com seus marcadores personalizados
Dica: Ao testar, use o botão "Testar Detecção" para verificar se o modelo e as informações são exibidos corretamente, independentemente do reconhecimento do marcador.

6. Implantação

Opção 1: GitHub Pages

  1. Crie um repositório GitHub para o projeto
  2. Faça o upload de todos os arquivos para o repositório
  3. Vá em Settings > Pages
  4. Selecione o branch main como fonte
  5. Clique em Save
  6. Seu site estará disponível em: https://seu-usuario.github.io/nome-do-repositorio/

Opção 2: Vercel

  1. Crie uma conta em vercel.com
  2. Instale o Vercel CLI: npm install -g vercel
  3. Na pasta do projeto, execute: vercel login
  4. Em seguida, execute: vercel
  5. Siga as instruções para concluir a implantação
Importante: Para que a AR funcione em dispositivos móveis, é necessário que o site seja servido via HTTPS. Tanto GitHub Pages quanto Vercel fornecem isso automaticamente.

7. Próximos Passos

Quando conseguir fazer o reconhecimento do primeiro planeta funcionar:

  1. Expanda para os outros planetas um por um, testando cada adição
  2. Adicione mais modelos 3D na pasta assets/models/
  3. Para vários planetas, você pode:
    • Criar um arquivo .mind único com todas as imagens de planetas
    • Ou criar arquivos .mind separados e alternar entre eles
  4. Personalize os estilos e informações para cada planeta
